Basics
This program loads W03_560 Machine object code into W03_560 Simulator.
To use this program, the user should be familiar with:
- W03_560 Machine architecture
- W03_560 Assembly object code

This document, in turn, will describe how to use this program and to understand its input and output. First, the user will learn how to install the program and get started. Second, the program input will be discussed in greater detail followed with a sample input file. Then, possible errorsare listed by their corresponding type, each one having a description. Finally, the sample output is shown.

Using the Linker/Loader

A brief overview on how to get the Linker/Loader to function

Usage

Before using the linker/loader it is important to have all the necessary files stored in the correct location and enough memory resources

Copying and Extracting

The Linker/Loader uses close to 1 Mb hard-drive space so before attempting to copy the files over, make sure there is enough space available. The directory name chosen is unimportant, so the zip-file should be copied over to the directory of your choice. Then you can unzip the zip-file with any extracting means and make sure that Loader.cpp, Loader.h, Loader2.h and Table.h are all within the same directory.

[top]

Compiling

Since the Linker/Loader was written in C++, it is important to use a C++ compiler for the next step. The compiler used in the testing was the GNU compiler. It is important that you are in the directory the assembled files are located and you type in the following command:

g++ -o Loader Loader.cpp

[top]

Running

Before running the Linker/Loader, it is absolutely necessary the input file(s) exist and follow the necessary format, which will be described in next section. The output file can be chosen at your discretion as long as you can remember the name of the file to view the output when you are done using the Linker/Loader. The initial loading address is also at your discretion. The following is the command line to use when using the Linker/Loader with the file names and initial loading address accordingly:

Loader [assembled_source_file(s)] <object file> initial_load_address

Valid Input Files

The most vital part for using the Linker/Loader is having a valid input file

Source File

The input file is the object file produced from the W03_560 Machine Assembler. The file should include an External Symbol table and the appropriate header and text records which are described below: (also a sample input file can be found here)

- Program Load Address (PLA)

 

- Segment Length

 

- M tag (if the program is relocatable)

- Location

 

- Data

- X tag (ONLY ifthe symbol is relocatable and external) followed by the symbol name

Limitations

- From the command line, the initial load address should be an appropriate value, not necessarily zero

- The input data must follow the structure below:
EXT Table
Header Record
Text Record

Error Messages

Error messages will be output when an invalid input has been given

Types of Errors

In the event of a record containing an error, an error listing will be output to the screen and compilation will halt. The object file will be created up to the point that the error occurred, This object file is INCOMPLETE and is not a valid input for the Simulator of the W03_560 Machine.
