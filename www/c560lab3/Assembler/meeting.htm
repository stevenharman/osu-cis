Group Over-Drive’s Meeting Minutes

Design Log

Group Over-Drive met 4 times together to discuss the assembler lab.  Those meetings, have been posted on the calendar below.  In addition, each design change was extensive so it is marked on the calendar but a detailed description is listed below. The Group did most of the work through online communication and a record of what was being changed and modified was logged.  The meeting minutes described below serve as a modification and design log.

Modification

February 7

Table.h file (which contains the Table class).

Table class for working with the various look-up tables.

Created Pass One, Intermediate, and Pass Two procedures.

Pass One working as far as creating symbol tables goes.

Created a correct literal table.

Added an Update_Values function to the Table class, so that it will put correct addresses in for all the literals.

The Change_Value function is completed.

February 16

Set up the intermediate file to receive the segment name, the initial program load address, the initial address for program execution, and the segment length.

February 19

Cleaned up the implementation of Pass_One.

February 20

function Is_Valid_Label(char* token) to make sure the label is legal (strlen(label) <=6, 1st char is alpha & rest are alpha/numeric).

CVS up and running.

Added/fixed the END psuedo-op so that if no operand is given, Pass_One can still figure out where the first executable address is and output it to the intermediate file.

Pass_Two working so that it is now receives the correct values for the R field, receives the correct values for the S(X) field, and is even adjusting them for the Initial_Load_Address offset.

Still seem to be having some trouble when ORI doesn't have a operand (the program is relocatable)

February 21

Set up the boolean for relocatable programs. Set up to output an 'M' for the Header Record (changed Pass_One to output an M for the initial load address if there's no operand), and working on setting it up to output an 'M' for each relocatable text record.

Changed Get_Data (now, you have to pass it a boolean value since the S field is what is modified in the case of a relocatable program). Seems to be working just fine (used the example program in class, and just removed the operand on ORI).
February 23

Added error checking in Pass Two for unknown instructions.

Added a check to make sure symbols can't be defined twice (in Pass One).

Changed cerr to listing in Pass One, and had the output include the invalid input (in relevant cases).

added a cerr << "Program exited abnormally" in the case of fatal errors in Pass One.

February 27

The results of the final testing process produced too many errors. The error checks were insufficient. The coding had to be changed to properly check for errors. Code was updated to pick up the necessary errors. Documentation had to be changed as well to update the testplan with the new code and errata section.
