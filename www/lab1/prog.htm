Implementation

The implementation of the simulator for the W03_560 Machine

Data Structure

The implementation of simulator is divided into four segments consisting of Simulator.cpp, Machine.h, Memory.h, and Processor.h.  The simulator is a C++ implentation of the W03_Machine.  This chapter will go into the data structure required by the simulator while the next will explain each segment

Memory

The memory designed in the simulator contains 256 addresses, or addresses 0-225.  The memory is structure is implemented in Memory.h and is in an array of integers.  The integer values of the memory are between the hex values 0x0 and 0xFFFF.  Seen below is the private declaration used for memory and each value within memory is a default halt value of 0xC0C0.

int mem_array[256] = 0xC0C0;

Registers

The simulator uses a four-element integer array to represent registers for data.  These registers can be used for indexing except for register 0.  Each register is 20 bits in length and any number outside this range is either fixed or could result in an error message being produced.

int reg_array[4];

Program Counter

The simulator uses a single integer value for the program counter.  This value is 20 bits in length and always-positive using 2’s complement.  This value always stores the address of the memory of the next instruction to be executed.

int P_Counter;

Design Modules

The Design Modules Used in the Simulator

Simulator.cpp

Simulator.cpp is the core file for the simulator it creates a machine type using Machine.h.  It is passed in four arguments from the command line, all four of which are file types, a check to see if four files are input is immediately performed before going any further.  Simulator.cpp declares the file streams, names them, and opens them accordingly to the format for the command line.  Simulator.cpp has five small tasks that it performs.

1)      Using Machine.h it calls Load_This and begins to get the necessary input for simulating

m.Load_This(main_input, outs, t_outs);

2)      Next, using Machine.h again, it calls Simulate_This to begin simulating the W03_560 Machine

m.Simulate_This(aux_input, outs, t_outs);

3)      After the final results are reached, dump the values of the registers

p.Dump_Regs(outs);

4)      In addition to dumping the values of the registers, the memory array is no longer needed as well

mem.Dump_Mem(outs);

5)      Provide tracing output to indicate where the program is in simulating

Machine.h

The Machine class encapsulates the W03_560 machine, creating the Memory and Processor objects upon instantiation and acting as a common link between them. In this way, it allows data to be passed between the elements of the machine, as well as keeping track of the number of instructions that are executed overall. In other words, the Machine class is the essential class, which keeps the basic functioning parts of the W03_560 machine within scope, and allows them to communicate with each other.

Private Operations

There are only 2 private operations in the Machine class, which are used exclusively by the operation Load_This (which is described in more detail below). These operations are used in tandem to take a string of characters which represents a hexadecimal value, and convert it to an integer which holds that hex value. It is useful to note that Char_To_Text is used by Text_To_Hex in order to find the values of the characters, which are shifted into their correct value places in the outgoing integer.

Machine::Char_To_Hex(char c)

                  Requires

                                  c is a hex digit in the form of a char

                Ensures

                                Return value = hex value of c

Machine::Text_To_Hex(char* str)

                  Requires

                                str is  a string of chars which represent hex digits

                Ensures

                                  Return value = hex value of str

Public Operations

While the requires clauses for Load_This and Simulate_This seem a bit light, it is important for the user (as well as the programmer) to note that, as indicated in the User’s Manual, if an invalid or incorrectly formatted main input file is passed to the W03-560 machine simulator, an error message will be produced, and the machine will never begin execution. Similarly, if an incorrect auxiliary input file is passed, Simulate_This will halt execution. (See the section of the User’s Manual on Error Codes for more information).

Machine::Load_This(fstream& main_input, fstream& main_output, fstream& trace_output)

                  Requires

                                main_input.is_open() and is an input stream and

                                main_output.is_open() and is an output stream and

                                trace_output.is_open() and is an output stream

                Ensures

                                The set of instructions coming in on the main_input stream will be loaded into machine memory.

Machine::Simulate_This(fstream& aux_input, fstream& main_output, fstream& trace_output)

                  Requires

                                aux_input.is_open() and is an input stream and

                                main_output.is_open() and is an output stream and

                                trace_output.is_open() and is an output stream

                Ensures

                                The set of instructions loaded into memory by Load_This and input provided by the auxiliary input file (if any) will be processed by the Processor Class as per W03-560 machine instruction execution parameters.

Processor.h

In addition to holding the array of integers that represents the registers of the W03_560 machine, the Processor class does the bulk of the work for the Simulator, as is suggested by the name of the function Processor::Do_This. A programmer who wants to understand or modify the inner workings of the machine would do well to understand the various operations and data structures that the Processor class makes use of in order to execute the instructions given to the simulator.

Constants

The private operations for Processor objects mainly do small but important and repetitive tasks. Due to their extensive use, however, it is necessary to understand their operation to some level of detail. It is also important to keep in mind that, because the Simulator has been written in C++, using its built-in 32-bit data type int extensively, many of these helper functions have no other purpose but to make the conversions that are necessary to correctly represent 20-bit values. Perhaps most interesting are the Is_Negative_Value and Leading_Ones functions, which test the lower 20 bits determine whether or not the value of an integer is negative in 20-bit two’s complement, and convert an integer to a 32-bit two’s complement integer, respectively. Conversely, there is the Leading_Zeroes function, which effectively converts said negative integer back to a form representable in 20-bit two’s complement.

                                  Return value #x|0xfff00000

                  Ensures

                                  Return value #x&0x000fffff

Public Operations

When considering calls made to them by other classes of the W03-560 machine simulator, the public operations of the Processor class are used just as frequently as the private operations. They are more in the forefront of the simulation of instructions, however. It may help to think of the private operations in terms of those which do non-trivial “behind-the-scenes” work, whereas the public operations cause actual changes to occur in data. They are indispensable for data access, both in terms of input and output. Because of the obvious nature of the workings of most of the public operations, as well as the fact that most of them are one or two lines of code at the most, the Do_This function will be concentrated on here. For more information on the other public functions, a simple glance at the source code should suffice.

 Processor::Do_This(Memory&, int OP, int R, int X, int S, fstream& aux_in, fstream& outs, fstream& t_outs)

                  Requires

                  0 <= OP <= 15 and

                0 <= R <= 3 and

                  0 <= X <= 3 and

                0 <= S <= 255 and

                  aux_in.is_open() and

                  outs.is_open() and

                t_outs.is_open()

                  Ensures

                [The required operation, as specified by OP, will be executed in accordance with  the instruction format of the W03-560 machine, returning a value of false after each instruction if instruction execution is to continue, and true if instruction execution is to halt after the current instruction.

Memory.h

The first and most important feature that Memory.h does is declare and initialize the memory array mem_array as shown below in private operations.  In addition to the public operations listed below, Memory.h uses 6 individual operations to parse the input text record into 6 different variables for later error checking and instruction handling.

Private Operations

Memory.h declares a private integer array to hold the values of the memory address.

int mem_array[256];

Public Operations

Memory.h uses four important public operations to perform the necessary tasks called upon it by Machine.h.

Memory::Load_Memory(int x, int y)

                  Ensures

                                  mem_array[int x] = int y;

Memory::Dump_Mem(fstream&)

                  Ensures

                                  outs << mem_array;

Memory::Get_Memory(int x);

                  Ensures

                                  return mem_array[int x];
